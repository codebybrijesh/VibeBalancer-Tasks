# Task 1 - Branches in Git

 What are Branches?

A branch in Git is a separate line of development. By default, Git has a main branch (usually called `main` or `master`). Branches allow you to create independent versions of your project.

Each branch acts like a copy of the project where you can make changes without affecting the main code.

 Why are Branches Useful?

Branches are useful in software development for the following reasons:

1. Safe Development  
You can work on new features or experiments without breaking the main project.

2. Parallel Work  
Multiple developers can work on different features at the same time using different branches.

3. Bug Fixing  
You can create a separate branch to fix bugs without disturbing ongoing development.

4. Easy Merging  
Once your work is complete, you can merge your branch back into the main branch.

5. Testing and Experimentation  
Branches allow you to test new ideas safely. If something goes wrong, you can simply delete the branch.


Conclusion

Branches help in organizing work, improving collaboration, and making development safer and more efficient.
